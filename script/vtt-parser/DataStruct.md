- # WebVTT 乐谱记号技术规范 v1.1（补充说明）

  本文档是对 v1.0 的精确化补充，明确数据类型边界、解析规则与异常处理。

  ------

  ## 1. 全局状态记号的值域与格式

  ### 1.1 `bpm=数值`

  - 数值范围：`20.0` ~ `400.0`（整数或小数，最多保留一位小数）。
  - 不符合范围的值：解析器应忽略该事件，并保持前一有效 BPM。

  ### 1.2 `key=调名`

  - 调名格式：`[a-gA-G](#|b)?`，如 `C`, `G`, `F#`, `Bb`。
  - 升降号表达：升号用 `#`，降号用 `b`（小写字母 b）。不支持 `+`/`-` 等符号。
  - 异名同调（如 `F#` / `Gb`）：视为两个不同的值，不进行等价转换，内部状态与写入值一致。

  ### 1.3 `dynamic=级别`

  - 标准枚举（由弱到强）：`pppp`, `ppp`, `pp`, `p`, `mp`, `mf`, `f`, `ff`, `fff`, `ffff`。
  - 忽略大小写（统一转为小写存储），超出枚举范围的字符串视为无效，保持前一有效力度。

  ### 1.4 `technique=名称`

  - 保留常见技法：`pizzicato`, `arco`, `con sordino`, `senza sordino`, `tremolo`（此处在技法语境下，不与第二节的震音混淆，解析器应根据上下文区分；也可直接废弃技法中的 tremolo，统一用持续效果类的 tremolo。规范建议：技法 `tremolo` 不接受，仅用持续效果类的 `tremolo`）。
  - 未知技法：解析器可安全忽略，不更新内部状态。

  ### 1.5 `expression=文字`

  - 自由文本，长度不超过 128 字符。
  - 不允许出现换行符，仅允许 ASCII 可见字符与中文、常用 Unicode 符号。

  ------

  ## 2. 持续效果记号的参数规范

  ### 2.1 `crescendo:to=` / `diminuendo:to=`

  - `to` 的值必须来自标准动态级别枚举（见 1.3）。
  - 起始力度为区间开始时当前全局力度值，结束力度为 `to`。
  - 区间外力度保持结束值。

  ### 2.2 `trill` / `tremolo`

  - `interval`：表示颤动/震动的音程。
    - `half`：半音（小二度）
    - `whole`：全音（大二度）
    - `minor3rd`：小三度
    - `major3rd`：大三度
    - 未指定时默认 `half`。
  - `speed`：以对应的音符时值粗略表示：
    - `slow`：约八分音符速度
    - `medium`：约十六分音符速度（默认值）
    - `fast`：约三十二分音符速度
    - 允许自定义数字，如 `speed=16` 表示十六分音符速度（具体映射由实现决定）。

  ### 2.3 `legato:voice=`

  - `voice` 为正整数，默认为 `1`。

  ### 2.4 `arpeggio:direction=`

  - `up`：由低音到高音
  - `down`：由高音到低音
  - `both`：先上后下
  - 未指定时默认 `up`。

  ------

  ## 3. 提示记号 `cue:text=` 的文本格式

  - 允许的字符：Unicode 基本多文种平面（BMP）内所有可见字符，包括中文、箭头（`→`、`↑`）、音名字母、特殊符号。
  - 禁止字符：换行符、回车符、竖线 `|`（保留用于未来扩展）。
  - 最大长度：256 个 UTF-8 字符，超出部分由前端截断。

  ------

  ## 4. 边界场景与解析规则

  ### 4.1 全局状态记号同时间点冲突

  - 若在同一 `start` 时刻出现多条同类型全局记号（如两个 `bpm=...`），以 **VTT 文件中最后出现的那条** 为准。
  - 产生多条的原因是编辑疏漏，解析器也可选择输出警告。

  ### 4.2 持续效果记号的重叠

  - 同类型效果（如两个 `crescendo`）若时间区间重叠：**不可叠加，后开始的事件覆盖先开始的事件**（即新的效果中断并取代旧效果）。
  - 不同类型效果重叠可共存（如同时有 `crescendo` 和 `trill`）。

  ### 4.3 时间区间异常

  - `start > end`：该事件非法，解析器应**忽略整条 cue**。
  - `start == end` 但承载的是持续效果记号（如 `crescendo:to=f`）：视为无效，因为持续效果需要正时长区间。解析器应忽略该事件。

  ### 4.4 未知类型或参数错误

  - 未知类型名：忽略该 cue。
  - 已知类型但参数缺失或无效（如 `crescendo` 无 `to`）：忽略该 cue。

  ------

  ## 5. 解析流程优先顺序

  1. 过滤非法时间区间。
  2. 按 `start` 排序，扫描事件。
  3. 对于全局状态事件，使用最后写入覆盖规则，维持当前状态快照。
  4. 对于持续效果事件，跟踪活跃效果列表，处理重叠时的取代规则。
  5. 对于提示事件，独立计算显示区间与倒计时。
